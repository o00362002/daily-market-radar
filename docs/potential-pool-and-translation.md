# 潛力候選池與繁中輸出

## 潛力候選池

潛力項目改為由事件內容特徵判斷，而不是由來源角色直接決定。

內容特徵包含：

- 試點、原型、沙盒、beta、概念驗證
- 新應用、Agent、API、開發者工具、自動化與整合
- 新商業模式、按量付費、零售媒體、代理支付
- 跨領域組合
- 採用、擴張、擴散、採購與新區域
- pilot-to-production、new-region、adoption-expansion 等 material delta

來源的 top-down / bottom-up 只作為輕量輔助特徵，不能單獨決定 Major / Potential。

`daily_push` 的資料層不再刪除超過每領域 3 筆的潛力候選。所有合格潛力項目都保留在 `RadarReportV2.items`；首頁僅每領域精選 3 筆，並以「完整潛力候選池」呈現其餘項目。完整日期報告仍顯示全部項目。

## 繁體中文輸出

- Deterministic 模式：系統產生的今日新增、不確定性、台灣說明與追蹤文字使用台灣繁體中文；外部新聞標題仍可能保留原文。
- API-assisted / auto-with-key：模型被要求只輸出台灣繁體中文，且不得改寫公司名、產品名、數字、日期或新增事實。
- 翻譯後的原始標題以 `原文標題：...` 保存於項目的 uncertainties，網站會另外顯示，方便回查。
- 翻譯失敗或沒有 AI key 時，系統保留原文並繼續產出，不讓翻譯能力影響報告完整性。

## 不變式

1. 潛力候選蒐集不以「太小、證據弱、非主流」為由預先刪除。
2. 今日報告只收新事件或具有 material delta 的既有事件。
3. 首頁精選只是 projection，不是資料刪除。
4. 翻譯不得創造來源中不存在的事實。
5. 原始來源與原文標題必須可回查。
