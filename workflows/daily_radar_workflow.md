# Daily Radar Workflow

Purpose: define the minimum daily execution path.

## Steps

1. Read entry files.
2. Load radar configs.
3. Load missed cases and watchlist.
4. Check recent reports for duplicates.
5. Search and verify sources.
6. Write the daily report from templates.
7. Add retry status and missed-case notes.
8. Record backtest or adjustment items.

## Output

Daily report in `reports/YYYY/YYYY-MM-DD.md`.

## Backtest

Backtest results belong in `reports/backtests/`.
