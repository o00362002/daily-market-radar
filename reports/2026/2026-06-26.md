## 6. 新增：零售全域搜尋模型修正

### 核心修正

每日播報的零售段落不應只問：

```text
今天有什麼百貨新聞？
今天有什麼品牌新聞？
今天有什麼服飾新聞？
```

而應先問：

```text
今天零售這個系統，哪一塊正在變？
是線上流量變了？
是線下場域變了？
是會員資料串起來了？
是百貨和品牌的關係變了？
是門市功能從銷售點變成體驗／履約／會員服務節點了？
```

### 新模型

```text
零售整體發展
├─ 線上通路
│  ├─ 官網 / DTC
│  ├─ 電商平台
│  ├─ 社群商務
│  ├─ 直播 / 內容電商
│  ├─ APP / 會員
│  ├─ LINE / 私域
│  └─ Retail Media
│
├─ 線下通路
│  ├─ 百貨
│  ├─ 購物中心
│  ├─ 街邊店
│  ├─ 旗艦店
│  ├─ 快閃店
│  ├─ 社區型店
│  ├─ Outlet
│  └─ 展會 / 市集
│
└─ 整合層
   ├─ OMO
   ├─ Omni-channel
   ├─ CDP / CRM
   ├─ 會員資料整合
   ├─ 庫存共享
   ├─ 門市取貨 / 門市出貨
   ├─ 線上看、線下試
   ├─ 線下體驗、線上回購
   ├─ LBS / 人流資料
   ├─ AI 導購
   └─ Personal Shopper / Clienteling
```

### 判斷重點

- 不同通路如何分工？
- 不同通路如何互補？
- 線上如何導流線下？
- 線下如何承接體驗並回流線上？
- 會員、庫存、訂單、推播、LBS、人流與 AI 導購是否正在整合？
- 對服飾零售、品牌進櫃、街邊展店、門市營運、庫存補貨、會員經營與商業化機會的影響是什麼？

## 8. 新增：街邊門市／街邊店固定子雷達

街邊店不應只是百貨段落的對照，而是零售線下通路的重要分支。

### 追蹤內容

- high street retail / street-level retail
- 路面店、街邊店、商圈店、獨立門市
- 旗艦店、概念店、品牌直營街邊店
- 社區型店、快閃街邊店、DTC 實體店
- 店租、空置率、租賃條件
- 商圈人流、城市更新、交通建設、觀光客回流
- 百貨與街邊店的通路分工

### 比較標準

| 面向 | 百貨／購物中心 | 街邊店 |
|---|---|---|
| 客流取得 | 有場域與活動人流 | 需品牌自帶流量或商圈流量 |
| 資料取得 | 百貨會員與場域資料較強，但品牌資料可能斷裂 | 品牌資料完整度較高 |
| 毛利與費用 | 抽成與規則較多 | 毛利較完整，但租金與導流壓力高 |
| 體驗彈性 | 受商場規範限制 | 品牌表現自由度高 |
| 風險 | 檔期、樓層、百貨策略影響大 | 商圈、租金、人流波動風險高 |

## 9. 新增：每日零售搜尋硬檢查問題

每日零售段落執行前，必須先問：

1. 今日線上通路發展是否改變顧客發現、比較、購買或回購路徑？
2. 今日線下通路發展是否改變百貨、購物中心、街邊店、旗艦店、快閃店或社區店的角色？
3. 今日是否出現線上與線下整合的新訊號，例如 OMO、CDP、LBS、庫存共享、BOPIS、門市出貨、AI 導購或會員資料整合？
4. 不同通路是否正在分工、互補、導流、轉換或承接？
5. 對服飾零售、品牌進櫃、街邊展店、門市營運、庫存補貨、會員經營與商業化機會有何影響？
6. 若無重大訊號，是否明確標示「已掃描但無重大變化」，而不是省略？

## 11. 本日結論

2026-06-26 的主要修正有兩層：

### A. AI 影片與短劇產業替代

- AI 新聞不能只看模型能力與發布。
- 必須追蹤是否進入實際產業流程。
- 對中國短劇、內容產業、影視後期與接案市場，需要加入簡體中文訊號源。
- 就業衝擊不可直接定論，除非有多來源證據。
- 若只有社群討論，應列為「候選訊號」，不可包裝成已證實新聞。

### B. 零售全域搜尋模型

- 零售段落不能只新增百貨或街邊子段落。
- 必須從「零售整體」出發，追蹤線上、線下與整合層如何共同構成零售。
- 百貨、街邊店、電商、社群、會員、LBS、Retail Media、CDP / CRM、OMO、AI 導購、庫存共享與商圈變化，都應被視為互相關聯的零售生態節點。
- 每日播報應從「找零售新聞」改成「掃零售生態系統」。
