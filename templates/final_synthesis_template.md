# Final Synthesis Template｜最後總和彙總與舊版比對

本檔定義每日市場情報報告最後必須輸出的總結段落。目的不是再摘要一次新聞，而是把所有雷達、指標、候選訊號、資料缺口與舊版播報差異整合成一個可回測的結論。

## 17. 舊版 / 新版播報比對與補漏

若同一天存在舊版播報、新版播報、手動補充播報或 reports/ 內歷史報告，最後必須比對：

| 來源 | 有抓到但另一版沒抓到的內容 | 屬於哪個雷達 | 證據等級 | 是否應補入今天結論 | 下一步處理 |
|---|---|---|---|---|---|
| 舊版播報 |  |  | 高 / 中 / 低 / 資料不足 | 是 / 否 |  |
| 新版播報 |  |  | 高 / 中 / 低 / 資料不足 | 是 / 否 |  |
| 使用者補充 |  |  | 高 / 中 / 低 / 資料不足 | 是 / 否 |  |

規則：

- 若舊版有抓到新版沒有，不能直接視為舊版較好，必須判斷它是「高價值漏抓」還是「低證據噪音」。
- 若新版有抓到舊版沒有，必須判斷是模型改善還是過度擴張。
- 若無法讀取舊版播報正文，必須明確寫「舊版播報無法讀取，無法完整比對」，不得假裝已比對。
- 使用者貼上的社群討論或連結可作為「使用者補充」，但必須標示未證實與來源。
