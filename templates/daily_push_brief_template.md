# Daily Push Brief Template

Daily Push Brief is a structured concise radar, not a free-form summary.

```text
Daily Push Brief 不代表可刪減結構。
Brief 只代表單則內容字數較短。
所有章節、欄位、證據追溯、台灣映射、指標狀態仍必須完整保留。
```

```text
Brief means:
- shorter wording per item
- fewer items than Full Daily Radar
- full structure preserved

Brief does NOT mean:
- removing required sections
- merging Taiwan mapping into generic paragraphs
- replacing news with conclusions
- treating indicators or synthesis as news
```

## 2. Domain Blocks

Each domain must include exactly:

```text
大型訊號：3 則
小眾候選：1 則
台灣映射：1–2 則
```

News / signals must be real events, data changes, company actions, policy changes, product releases, market moves, or source-backed observations.

Indicators, synthesis, themes, and model conclusions must not be counted as news.

Each news item should keep concise evidence trace:

```text
ID：
事件：
來源 / 時間：
證據等級：high / medium / low / insufficient_data
不確定點 / 下一步：
```
